### mlRecommend orchestration order (Fable-corrected)

3. **lineage FIRST, then intent** (Fable #1 ‚Äî CRITICAL): if `parentSnapshotId`: re-read parent by
   `{_id,user}` ‚Üí **404** if absent/cross-user; `intent := parent.intent`; derive child Lens VERBATIM
   from parent row (occasion/weather/weatherRaw/location/forcedItemId/seedDate/constraints);
   `generationIndex = parent+1`; `controls = normalizeControls(body.controls)`. Else (root):
   `generationIndex = 0`; `intent = body.forcedItemId ? "rescue_item" : "daily"`; build Lens from body
   (occasion + resolved weather bucket + weatherRaw + location + forcedItemId + seedDate=today());
   `controls = {lockedItemIds:[],dislikedItemIds:[]}` and **non-empty body.controls ‚Üí 400 root-controls**
   (Next mirror; the plan says contract_invalid ‚Äî reconcile to a stable 400 like locked‚à©disliked, note it).
   [Rescue re-roll breaks if intent is computed from body before lineage ‚Äî that was the #1 bug.]
4. mint `snapshotId = deps.newId()`.
5. idempotency early read-check: `findOne({user,requestId})`; if exists & `identityMatches` ‚Üí
   **replay** winner via `projectBrowserResponse(winnerDoc)`, no service call; if exists & mismatch ‚Üí
   **409 request_id_conflict**.

   countDocuments({user})` **computed ONCE** (Fable Q2 ‚Äî reused for wire + merge). Preflight mirrors

  field-for-field (when carried); swapped-body mutant rejected.

## Two spec-touching reconciliations (fix docs in same pass ‚Äî "conflicts are bugs")
1. **Weather in G5 (Fable #2).** Keep the plan's G5 set (weather/weatherRaw included). The promise
   (idempotent retry = SAME render) REQUIRES the whole render input frozen at the Generate action incl.
   weather; so weather must be resolved-once-and-frozen (C6 F10 lensSummary), NEVER live-refetched on a
   retry. Add a ¬ßF/¬ßC.4 pin: "weather resolved once per action + frozen in the F10 envelope; the route
   uses the client-carried resolved bucket/weatherRaw on a retry, never re-fetches." (If instead weather
   were live-refetched, it would false-409 the F10 loop ‚Äî so the freeze is load-bearing, not optional.)
2. **HTTP status map / preflight mirror.** Next mirrors the CHEAP request-decidable preflights ‚Üí
   stable 400 (`locked‚à©disliked`, `forced‚ààdisliked`, root-controls) so the client gets a distinct,
   stable error instead of a generic degraded. Structural-feasibility + control-id-in-wardrobe stay
   service-side. Status map: 401/404 auth; 404 forged parent; 400 malformed requestId / the 3 mirrored
   preflights; 409 request_id_conflict / forced_item_unavailable; 200+degraded body for service outage
   (service_unavailable/rate_limited/auth_failed) AND contract_invalid (adapter reject / service
   contract_invalid / payload cross-check fail); 200+normal for success incl. healthy-empty nSurfaced=0.
